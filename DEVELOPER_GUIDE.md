# Developer Guide

## Tech Stack
*   **Framework:** [Astro](https://astro.build/) (v5)
*   **Theme/Doc Engine:** [Starlight](https://starlight.astro.build/)
*   **Styling:** [Tailwind CSS](https://tailwindcss.com/) (integrated via Vite)

## Project Structure
*   `src/content/docs/`: Markdown/MDX content files.
*   `src/assets/`: Static assets (images, logos).
*   `src/styles/`: Custom CSS files (`custom.css`).
*   `astro.config.mjs`: Main configuration file.
*   `tailwind.config.mjs`: Tailwind configuration (if present/needed, currently using Vite plugin).

## Configuration (`astro.config.mjs`)
The `sidebar` property in `astro.config.mjs` controls the main navigation menu.

### Autogenerated Groups
For sections like "Knowledge Base", we use `autogenerate` to automatically list all files in a folder:
```js
{
    label: 'Knowledge Base',
    autogenerate: { directory: 'knowledge-base' },
},
```
This means any new file added to `src/content/docs/knowledge-base/` will automatically appear in the sidebar.

### Manual Links
For single pages like "About", we use manual entries:
```js
{
    label: 'About',
    slug: 'about',
},
```

## Customization

### Styling
Global styles are defined in `src/styles/custom.css`. You can use standard CSS or Tailwind utility classes in your components/pages.

### Components

#### SubsectionPreview
We use a custom `<SubsectionPreview />` component to automatically generate content preview cards.
- **Path:** `src/components/SubsectionPreview.astro`
- **Usage:** `<SubsectionPreview directory="youtube" />`
- **Features:**
  - Automatically fetches content from specified directory
  - Sorts by `publishDate` (newest first) with fallback to title
  - Displays first 10 items (pagination)
  - Extracts YouTube video IDs for thumbnails
  - Shows gradient separator lines between cards
  - Horizontal card layout: thumbnail left (40%), content right (60%)
  - Date badge and responsive design
- **Performance:** Static generation, no client-side JavaScript

## Security & Best Practices
*   **Dependencies**: Regularly run `npm audit` to check for vulnerabilities.
*   **Secrets**: Ensure no sensitive keys (API keys, tokens) are committed. Use environment variables if needed.
*   **Git**: The `.gitignore` is configured to exclude sensitive files like `.env` and `.DS_Store`.
*   **Static Site**: Being a static site, there is no backend surface area, which enhances security.

## Mobile Verification
The site is designed to be fully responsive.
*   **Viewport**: Tested on standard mobile resolutions (e.g., iPhone X: 375x812).
*   **Navigation**: The sidebar collapses into a hamburger menu on mobile.
*   **Layout**: `SubsectionPreview` and other grids switch to a single column layout on smaller screens.
